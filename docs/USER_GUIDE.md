# 智能云台拍照助手 — 用户使用指南

## 1. 安装与准备

首先，建议准备好具有 Python 3.9+ 环境的电脑：

```powershell
# 创建虚拟环境
python -m venv .venv
.\.venv\Scripts\Activate.ps1
# 安装项目依赖 (Mediapipe, OpenCV, Ultralytics, Pillow 等)
pip install -r requirements.txt
```

**强烈建议：配置 SiliconFlow 云端 AI 大模型能力**

在运行程序之前的命令行配置以下环境变量（或配置系统环境变量），享受基于视觉大模型的超强摄影助理能力：

```powershell
$env:SILICONFLOW_API_KEY="your-key"
$env:SILICONFLOW_MODEL="Pro/OpenGVLab/InternVL2.5-73B"
```

## 2. 启动程序

程序移除了过时的纯命令行版本，目前全量基于高效 GUI 图形交互。

```powershell
# 连接同一局域网内手机摄像头，且使用硬件云台（串口比如 COM10）
python main.py --stream-url "http://192.168.1.6:4747/video" --bus-serial-port COM10 --bus-baudrate 115200

# 本地无硬件调试（调皮计算机本地摄像头0，模拟云台）
python main.py --stream-url 0 --mock-gimbal
```

## 3. 核心界面布局

| 区域 | 构成 | 功能说明 |
|------|------|----------|
| **实时视频** | 界面核心 (左上) | 实时抓取视频流，叠加强大的 MediaPipe+YOLO 视觉辅助框和跟随骨架 |
| **滚动控制面板** | 右侧 (全景) | 集全参数微调、三项控制模式跳转、显示覆盖、模板挑选与 AI 处理面板 |

## 4. 三大核心工作模式

本系统通过右侧的“模式”大类无缝切换。

1. **手动** (Manual): 右侧方向键、回中键接管整个云台进行手工定位拍摄。
2. **自动跟随** (Auto Track): 全局搜索目标人脸和高置信度人物，云台将人物锁定在特定追踪点（如“肩部”或“人脸”），跟随速度可通过 UI 滑块即时切换（偏快、匀速等）。
3. **模板引导** (Smart Compose): 用户上传具有标准拍摄参考价值的图片（模板），跟随引擎计算人物画幅中心、大小比及动作姿势匹配度并智能转动打分。

## 5. 最强 AI 指导功能

系统内置 SiliconFlow API 批量智能调用（极简、高效 Token 优化机制）。主要按钮操作：

### 5.1 上传独立照片评定
* **上传评分**: 系统要求上传一张本地照片进行评分，返回修改建议。
* **背景分析(上传)**: 分析您上传的空背景/风景图，由 AI 推荐拍摄的黄金分割站位以及当前光线的调整策略。
* **模板+背景双向指导**: 同时上传参考姿态与目标背景环境进行组合评估，测试是否“完美复刻”与落地指导。

### 5.2 全自动 AI 云台接管
以下按键启动之后，程序会主动驱动云台执行大量数据收集：
* **智能多张优选（AI 自动找角度）**: 启动后，云台快速转圈收集被摄者多个微调角度的照片（受“扫描候选数量”滑块调节，推荐 2~9 张），一次性发送大模型过滤最佳表现瞬间，云台**自动定位回表现最佳处**并存储一张带有 `_AI分析最佳结果` 的完美出图。
* **背景雷达探测（现场锁机位并等待人物）**: 选择后系统驱动云台全面探测记录现场风景的所有可用背景帧，将这些空旷的背景发送 AI。判断哪一张对被摄人最佳，转到该位置后彻底“锁云台”，被拍者只需要站进系统提示的黄金框内触发拍照。
* **解除机位锁定**: 从上一项锁定功能紧急切回普通追踪能力。

## 6. 参数调优微操手册

右侧的「跟随调参」配置滑块均为实时生效，建议了解：

### A. 灵活抓拍动作判定
| 控制滑块 | 效用 / 建议 |
|----------|-------------|
| **手势倒计时** | 张手-握拳后的预留时间。建议调节为 3.0 ~ 4.0 秒供人物整理表情。 |
| **手势稳定帧 / 最短保持秒** | 控制误触率。建议握拳动作判定至少保留 10 帧以上。单手张开最少 0.3 秒判定。 |

### B. AI 智能扫描定制
结合 AI 限制，特意为您准备的高级组合拳（批量控制）：
| 控制滑块 | 效用 / 建议 |
|----------|-------------|
| **扫描范围/步长** | 获取不同空间内的候选背景范围大小。默认建议 2° - 5°。 |
| **候选数量 (建议2~9张)** | 批量收集供一次 API 判定最优解上限，极速出图。过多将延长大模型等待期。 |
| **扫描候选等待** | 控制云台每次转动到新角度后的相机防抖停留时间，支持 0.5s ~ 3.0s。 |
| **背景抓取延时** | 发起锁定云台操作前，给予摄影师出画的提示时间。默认为 1s。 |

## 7. 照片防丢失及命名

所有的拍摄动作均存放在当前目录的 `captures/YYYY-MM-DD` 目录下。文件名自携标识记录照片生成方式与时机，如 `142718_强制手势拍照_xyz.jpg` 或 `143000_AI分析最佳结果_xyz.jpg`。
所有的被保存照片均会调用 AI 后台处理评分，评分提示将会直达最右下方的 “AI 聊天区”。

## 8. 使用疑难说明
* **日志突然闪过 Exception (例如 Failed / RuntimeError):** 极有可能是当时的网络无法访问 AI 大模型或者由于光线差遮住了摄像头采集的人影。您可以看左下角系统日志查看 `str(exc)` 给出的根本原因。
* **大模型一直等待不回信息:** 这是由于请求排队时间长，我们在内部封装了最长达 45 秒以上的 API 轮询机制自动规避失效，如长期不管用，重调模型参数或选择更加强大的 Siliconflow Model 系列。
